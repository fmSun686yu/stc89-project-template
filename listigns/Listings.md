# listings 文件夹 ——— Listings

## listings 文件夹的作用:
listings 文件夹中包含编译器和汇编器生成的“清单（Listing）文件”，主要用于调试、分析和优化程序。

## 包含什么内容？
这些文件通常以 .lst 或 .map 结尾。
一般包含：

### 1. 汇编清单（Assembly Listing，.lst 文件）
**包含：**
- C 语言源码
- 对应的汇编代码（反汇编）
- 每条指令的地址
- 编译器优化后的实际执行指令
- 变量、寄存器使用情况

**用途：**
- 查看 C 代码到底被编译成了什么指令
- 优化程序速度和大小
- 定位死循环、跳转错误
- 调试与 Bootloader、精确时序相关的问题

### 2. 链接器地图文件（Linker Map，.map 文件）
**包含：**
- 所有变量、函数的内存地址（ROM、RAM 分配情况）
- 各段（code/data/xdata/bdata/bit 等）的大小
- 程序总的 flash 占用、RAM 占用

**用途：**
- 检查内存分布是否合理
- 找到占用空间最大的函数或数组
- 分析 RAM 溢出、堆栈冲突

### 3. 编译过程的详细输出记录
**有时还包含：**
- 编译器优化等级信息
- Warning/Info 的详细说明
- 寄存器分配情况

## 总结一句话
listings 是 Keil 自动生成的“编译与链接详细报告”文件夹，用来查看 C 代码对应的汇编指令、内存分布、变量地址等，帮助进行调试和性能优化。
